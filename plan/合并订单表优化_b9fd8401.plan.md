# 合并 order_items 到 orders 表

## 背景

当前 `orders` 和 `order_items` 是一对多关系，但业务上一个订单只有一个 SKU。`order_items` 的独立存在导致大量不必要的 JOIN 和子查询。合并后可以：

- 消除所有 `order_items` 的 JOIN/子查询
- 简化 API 返回结构（无需分开返回 `order` + `items`）
- 减少数据库 IO

## 合并的字段

将 `OrderItem` 的以下字段并入 `TicketOrder`：

- `sku_id` (uint) - 门票SKU ID
- `sku_name` (string) - SKU名称
- `price` (float64) - 购买价格
- `quantity` (int) - 购买数量
- `visit_date` (time.Time) - 游玩日期

## 需要修改的文件

### Server 端

**1. 模型层**

- [server/plugin/ticket/model/order.go](server/plugin/ticket/model/order.go) - 新增 5 个字段（SkuID, SkuName, Price, Quantity, VisitDate），新增 `ProductName` 虚拟字段（`gorm:"-"`）
- [server/plugin/ticket/model/order_item.go](server/plugin/ticket/model/order_item.go) - 删除整个文件
- [server/plugin/ticket/model/request/mini_order.go](server/plugin/ticket/model/request/mini_order.go) - 将 `Items []MiniOrderItem` 扁平化为单个 item 的字段（SkuID, Quantity, VisitDate 直接放到 `MiniOrderCreate` 中），删除 `MiniOrderItem` 结构体

**2. 服务层** - [server/plugin/ticket/service/order.go](server/plugin/ticket/service/order.go)

这是改动最大的文件，需要：

- 删除 `GetMaxVisitDateByOrderIDs` - 不再需要子查询，`visit_date` 直接在 `orders` 表上
- 删除 `GetSkuNamesByOrderIDs` - `sku_name` 直接在 `orders` 表上
- 删除 `GetProductNamesByOrderIDs` - 改为直接 JOIN `ticket_sku` + `ticket_products`，基于 `orders.sku_id`
- 删除 `GetScenicImageByOrderIDs` - 同上，改为直接从 `orders.sku_id` JOIN
- 删除 `fillOrderItemsProductName` - 不再需要
- 简化 `GetList` - 待核销过滤中的子查询 `(SELECT MAX(visit_date) FROM order_items WHERE ...)` 改为直接 `WHERE visit_date >= ?`
- 简化 `GetByID` - 不再需要单独查 `order_items`，直接返回一个 `TicketOrder`
- 简化 `GetByOrderNoPublic` - 同上
- 简化 `CreateOrder` - 不再循环 items，直接处理单个 SKU，字段直接写入 `order` 对象，一次 `Create` 即可
- `OrderStatusLabel` - 签名简化，`maxVisitDate` 可直接从 order.VisitDate 取

**3. API 层**

- [server/plugin/ticket/api/order.go](server/plugin/ticket/api/order.go)
  - `Find` / `GetOrderByCodePublic` / `VerifyOrderByCodePublic` - 不再返回 `{order, items}` 分开的结构，改为直接返回 order 对象（或保持 `{order}` 包裹）

### 数据迁移

需要手动执行一段 SQL 将已有 `order_items` 数据回填到 `orders` 表（GORM AutoMigrate 只加列不迁数据）：

```sql
-- 先由 GORM AutoMigrate 自动加列后执行
UPDATE orders o
JOIN order_items oi ON oi.order_id = o.id
SET o.sku_id = oi.sku_id,
    o.sku_name = oi.sku_name,
    o.price = oi.price,
    o.quantity = oi.quantity,
    o.visit_date = oi.visit_date
WHERE o.sku_id IS NULL OR o.sku_id = 0;
```

确认数据迁移完成后，可手动 `DROP TABLE order_items;`。

## 注意事项

- 如果未来可能需要支持一个订单多个 SKU，则不建议合并；但如果业务确定一单一 SKU，合并是合理的
- 小程序端下单接口的请求体结构会变（从 `{items: [{skuId, quantity, visitDate}]}` 变为 `{skuId, quantity, visitDate}`），需要同步更新小程序代码
